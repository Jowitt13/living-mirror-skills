# 自我蒸馏框架 v2.0 + v0.4/v0.5 扩展 + v0.6 动态镜像（统一版本）

> 框架版本边界：自我蒸馏系统的统一核心框架版本是 **v0.6**。  
> Skill Pack v0.7-v0.9 只提供入门、隐私、复核、行动、导出和社区模板等操作层能力，不改写 manifest、自画像或权威框架文件里的 `framework_version: "v0.6"`。

## 定义

从聊天记录、flomo、日记、语音转录等碎片记录中，蒸馏出此刻的用户：一套带证据、带置信度、允许矛盾、可被推翻、可回滚的自画像。

Living Mirror 不是人格克隆，也不是让 AI 模仿用户说话。它是一面可验证、可修正、会迭代的镜子。

## 五层架构

1. 操作辅助层（Skill Pack v0.7-v0.9）：冷启动访谈、数据体检、同意范围、隐私等级、复核队列、行动转译和导出模板。
2. 数据采集层：把聊天记录、flomo、日记、语音转录等数据统一为碎片格式，保留来源、时间戳、sender、会话、内容类型。
3. 蒸馏框架层：用 Part A 自我记忆 11 维度、Part B 人格 5 层，以及证据分级、CONFLICT、Correction、版本管理四套机制进行分析。
4. 复核与行动层：生成复核队列，把已确认洞察转成 7 天实验、沟通脚本、边界脚本或关系地图。
5. 输出层：生成自画像、批次报告、主题报告、conflicts、corrections、changelog、manifest、actions、reviews 和 exports。

## 统一碎片格式

```json
{
  "id": "frag-YYYYMMDD-001",
  "source": "wechat | flomo | journal | voice_transcript | other",
  "source_meta": {
    "platform": "wechat",
    "conversation": "可选会话名",
    "raw_file": "原始文件路径"
  },
  "timestamp": "YYYY-MM-DDTHH:mm:ss",
  "sender": "me | other | system | unknown",
  "text": "脱敏后的内容或原始文本",
  "content_type": "text | voice_transcript | image_caption",
  "dimension_hint": ["价值观与情绪"]
}
```

## Part A：自我记忆 11 维度

基础 5 维度来自 v2.0 框架；v0.4 增加 5 个生活/思维/偏好维度；v0.5 增加 1 个可选敏感维度。

1. 价值观与情绪
2. 行为与决策模式
3. 人际关系
4. 目标与方向
5. 个人经历
6. 饮食偏好
7. 生活习惯
8. 思维方式
9. 世界观与人生观
10. 爱好系统
11. 亲密/性表达（可选敏感维度，必须获得用户明确同意）

第 16 主题涉及亲密或性表达时，必须遵守更严格的边界：

- 只在用户明确要求或明确授权时分析。
- 可以完全跳过，不影响自画像主流程。
- 不输出刺激性细节，不做道德评判，不把偏好病理化。
- 优先用抽象模式描述，如边界、亲密脚本、表达方式、压力影响、关系语境。
- 任何原文引用都要克制、脱敏，并验证 sender。

## Part B：人格五层

1. Layer 1 硬规则：不可违背的底线、稳定行为规则、雷区。
2. Layer 2 身份：用户是谁、当前阶段、核心自我认同。
3. Layer 3 说话风格：句式、语气词、不同关系中的表达差异。
4. Layer 4 情感模式：触发器、调节方式、压力/亲密中的变化。
5. Layer 5 人际行为：关系中的行动方式、冲突处理、照顾/依赖模式。

## v0.6 动态镜像规则

v0.6 将前面的动态判断统一成正式版本。Living Mirror 不只问"这个人是什么样"，还要问"这个判断在什么情境下成立"。写入自画像前，重要洞察需要经过动态判断：

1. 区分 `stable_trait`、`stage_state`、`special_period_response`、`relationship_triggered`、`pending_pattern`。
2. 标注时间、关系、身体状态、环境、事件、媒介等情境权重。
3. 将置信度拆成 `evidence_confidence`、`interpretation_confidence`、`stability_confidence`。
4. 分离事实、解释和临时命名，避免一个好听的标签跑在证据前面。
5. 为重要洞察维护 `CE-XXX` 反证索引，记录例外、相反证据和替代解释。
6. 写出可推翻条件：什么新证据、用户纠正或替代解释会削弱这个洞察。
7. 优先使用用户自己的语言命名模式；理论标签只能作为辅助。
8. 使用 `insight-schema-v0.6.json` 约束核心结构化洞察；`insight-schema-v0.9.json` 只作为 Skill Pack 产品层扩展字段参考。
9. 使用 `quality_check_distillation.py` 自动检查自画像或主题报告。

详细规则见 `dynamic-mirror-rules.md`。

## Skill Pack v0.7 入门与信任层

Skill Pack v0.7 解决用户开始前的真实阻碍：

- 没有整理好数据：使用 `light_start` 和冷启动访谈。
- 数据很杂：运行 `scripts/diagnose_distillation_inputs.py`。
- 不确定能不能分享：先定义 private / shareable / public 隐私等级。
- 不想分析某些关系、时期或主题：写入 consent scope。
- 想删除某个来源：按遗忘/删除规则把相关洞察降级为待复核。

细则见 `onboarding-and-data-diagnosis.md` 和 `privacy-consent-redaction.md`。

## Skill Pack v0.8 行动与关系层

Skill Pack v0.8 让自画像不止停在“说得很准”：

- 用 `make_review_queue.py` 生成用户复核队列。
- 用 `action-translation.md` 把洞察转成 7 天实验、沟通脚本、边界脚本、修复脚本、决策镜头或环境调整。
- 用 `relationship-pack.md` 生成关系地图、冲突循环、修复地图。

低稳定性的洞察不能直接变成重大行动，只能变成小而可逆的实验。

## Skill Pack v0.9 产品化与社区模板层

Skill Pack v0.9 让 v0.6 框架可以被更多人 fork、展示和贡献：

- `assets/templates/` 提供冷启动访谈、行动卡、关系地图、公开案例、Obsidian index 等模板。
- `community-template-kit.md` 定义社区模板的结构和贡献标准。
- 公开输出必须先按 `privacy-consent-redaction.md` 脱敏。

## 两轮蒸馏

### 第一轮：按时间段蒸馏

适合首次建档或数据跨度很长的用户。按季度、年份、人生阶段或用户指定时间范围切分，每个批次产出一份报告。

批次报告必须包含：

- 特殊时期标注
- 数据概览
- 长消息精选
- 深夜/高情绪对话
- 关键词命中，只做导航
- 关键事件时间线
- 会话画像
- 阶段性洞察
- CONFLICT 候选
- 蒸馏器自评

### 第二轮：按主题纵向蒸馏

最多 16 个主题各跨全时段分析；可根据用户目标裁剪主题，第 16 主题必须明确授权后才启用：

1. 价值观与情绪
2. 行为与决策模式
3. 与伴侣关系
4. 与家人关系
5. 与朋友关系
6. 目标与方向
7. 个人经历时间线
8. 说话风格演变
9. 冲突处理进化
10. 消费观演变
11. 饮食偏好
12. 生活习惯
13. 思维方式
14. 世界观与人生观
15. 爱好系统
16. 亲密/性表达（可选敏感主题，必须获得用户明确同意；可跳过）

每个主题流程：

1. 用关键词和时间范围筛选候选消息。
2. 读取原文和上下文，验证 sender。
3. 按时间纵向排列，识别模式、变化、矛盾。
4. 提取洞察，每条附证据等级、sender 验证、置信度。
5. 标注特殊时期影响。
6. 与上一版本和第一轮批次洞察对比。
7. 交给用户复核。

## 合并原则

两轮结果合并不简化。以已完成的纵向主题为结构骨架，把时间线批次中的阶段深度、verbatim 证据、演变细节编织进去。不要为了形成好看的结论而压扁矛盾。

如果用户只完成 10 个基础主题，就以 10 主题为骨架；如果完成 v0.4/v0.5 扩展，就以 15 或 16 主题为骨架。扩展主题只能增加分辨率，不能覆盖用户本人对自己的修正。

## 用户复核节点

每个主题报告完成后暂停，等待用户复核：

- 洞察不对：立即写入 Correction。
- 洞察需要补充：追加证据和修订说明。
- CONFLICT 可以关闭：标记为已解决，并说明关闭依据。
- 敏感主题不想继续：立即停止该主题，不追问。

用户的裁决高于蒸馏器判断。

完成初稿后，运行：

```bash
python scripts/make_review_queue.py --input self-portrait-YYYY-MM.md --output reviews/review-queue-YYYY-MM.md
```

## 可选输出形态

除完整自画像外，可根据用户需求输出：

- 短版自画像：5 到 9 条带证据的要点。
- 关系地图：按关系类型或经同意的具体关系整理互动模式。
- 变化时间线：标记一个模式何时出现、增强、变弱或被 Correction 推翻。
- 验证问题清单：把低置信度洞察变成可供用户确认、否定或重命名的问题。
- 证据账本：并排呈现支持证据、反证、当前置信度和状态。
- 情境权重概览：说明哪些时间、关系、身体状态、环境或事件正在影响自画像。
- 用户语言词典：保存用户自己的命名，把外部理论标签放在次要位置。
- 7 天行动实验：把确认过的洞察转成小而可逆的行动。
- 公开脱敏稿：用于 GitHub、博客、Slides、小红书等场景。
