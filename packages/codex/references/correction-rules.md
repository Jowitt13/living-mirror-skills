# Correction 规则

## 核心原则

用户说"我不是这样"时，立即修正。蒸馏器不是终审。不要删除原洞察，要保留判断如何被推翻的痕迹。

## 6 类已验证教训

1. 特殊时期误判：考试周、期末周、假期等不能代表稳态。
2. 关键词定性误判：词频不能直接判断关系、情绪或依恋类型。
3. 时间情境误判：暑假无学业活动不能推断学业被边缘化。
4. sender 归属错误：把 A 做的事误判成 B 做的事。
5. 角色归属错误：把辅导者/被辅导者、照顾者/被照顾者搞反。
6. 核心论点归属错误：把用户自我暴露误判成对方表达。

## corrections.md 模板

```markdown
# 纠正记录

> 用户说"我不是这样"，立即修正。纠正不删除原洞察，保留迭代痕迹。

---

## CORR-001  YYYY-MM-DD
- 被推翻的洞察：<原洞察标题或编号>
- 推翻理由：<用户指出的问题>
- 证据（用户提供）：[verbatim/artifact] <证据>
- 修正后表述：<新的、更精确的表述>
- 影响范围：<自画像章节/主题报告/CONFLICT>
- 已同步至：<文件名或待同步>
- 蒸馏器反思：<这次错误暴露了什么规则缺口>
```

## 执行规则

- Correction 立即生效，不等月度合并。
- 原洞察标注"已被 CORR-XXX 推翻"，不要删掉。
- 下一次月度蒸馏前先读取所有 Correction。
- 同类错误重复出现时，把错误上升为新规则，写入 SKILL 或 reference。
- 用户自我报告优先于统计指标。
