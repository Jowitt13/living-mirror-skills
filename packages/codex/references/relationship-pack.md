# Relationship Pack v0.8

Use this reference when the user wants to understand partner, family, friend, workplace, or group patterns.

The relationship pack studies the user's side of interaction and the pattern between people. It must not pretend to know another person's inner life without evidence and consent.

## Relationship Modes

| Mode | Use when | Output |
|---|---|---|
| `relationship_overview` | The user wants a map across partner/family/friend/work. | Relationship role map and recurring triggers. |
| `specific_relationship` | The user names one relationship and consents to analyze it. | Timeline, conflict loop, repair map, validation questions. |
| `repair_focus` | The user wants help after conflict or silence. | Repair script and non-repair boundary. |
| `boundary_focus` | The user feels drained, pressured, or unclear. | Boundary pattern and scripts. |
| `consent_safe_public` | The user wants to share relationship insights publicly. | Anonymized pattern-only artifact. |

## Do Not Overreach

- Do not diagnose the other person.
- Do not infer motives from a single quote.
- Do not turn one fight into a relationship identity.
- Do not encourage surveillance, pressure, or manipulation.
- Do not publish identifiable third-party material.

## Validation Questions

Ask the user:

1. Is this pattern specific to this relationship, or does it appear elsewhere?
2. Which line sounds like your own language, and which feels imposed?
3. What would the other person probably disagree with?
4. What evidence would make this pattern narrower?
5. What should not be included in any shareable version?
