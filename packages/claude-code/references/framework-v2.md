# 自我蒸馏框架 v2.0

## 定义

从聊天记录、flomo、日记等碎片记录中，蒸馏出此刻的用户：一套带证据、带置信度、允许矛盾、可被推翻、可回滚的自画像。

## 三层架构

1. 数据采集层：把聊天记录、flomo、日记等数据统一为碎片格式，保留来源、时间戳、sender、会话、内容类型。
2. 蒸馏框架层：用 Part A 自我记忆 5 维度、Part B 人格 5 层，以及证据分级、CONFLICT、Correction、版本管理四套机制进行分析。
3. 输出层：生成自画像、批次报告、主题报告、conflicts、corrections、changelog、manifest。

## 统一碎片格式

```json
{
  "id": "frag-YYYYMMDD-001",
  "source": "wechat | flomo | journal | other",
  "source_meta": {
    "platform": "wechat",
    "conversation": "可选会话名",
    "raw_file": "原始文件路径"
  },
  "timestamp": "YYYY-MM-DDTHH:mm:ss",
  "sender": "me | other | system | unknown",
  "text": "脱敏后的内容或原始文本",
  "content_type": "text | voice_transcript | image_caption",
  "dimension_hint": ["价值观与情绪"]
}
```

## 两轮蒸馏

### 第一轮：按时间段蒸馏

适合首次建档或数据跨度很长的用户。按季度、年份、人生阶段或用户指定时间范围切分，每个批次产出一份报告。

批次报告必须包含：

- 特殊时期标注
- 数据概览
- 长消息精选
- 深夜/高情绪对话
- 关键词命中，只做导航
- 关键事件时间线
- 会话画像
- 阶段性洞察
- CONFLICT 候选
- 蒸馏器自评

### 第二轮：按主题纵向蒸馏

10 个主题各跨全时段分析：

1. 价值观与情绪
2. 行为与决策模式
3. 与伴侣关系
4. 与家人关系
5. 与朋友关系
6. 目标与方向
7. 个人经历时间线
8. 说话风格演变
9. 冲突处理进化
10. 消费观演变

每个主题流程：

1. 用关键词和时间范围筛选候选消息。
2. 读取原文和上下文，验证 sender。
3. 按时间纵向排列，识别模式、变化、矛盾。
4. 提取洞察，每条附证据等级、sender 验证、置信度。
5. 标注特殊时期影响。
6. 与上一版本和第一轮批次洞察对比。
7. 交给用户复核。

## 合并原则

两轮结果合并不简化。以 10 主题为结构骨架，把时间线批次中的阶段深度、verbatim 证据、演变细节编织进去。不要为了形成好看的结论而压扁矛盾。

## 用户复核节点

每个主题报告完成后暂停，等待用户复核：

- 洞察不对：立即写入 Correction。
- 洞察需要补充：追加证据和修订说明。
- CONFLICT 可以关闭：标记为已解决，并说明关闭依据。

用户的裁决高于蒸馏器判断。
