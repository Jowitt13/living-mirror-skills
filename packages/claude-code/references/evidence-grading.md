# 证据分级机制

## 三级证据

| 等级 | 用法 | 权重 |
|---|---|---|
| `verbatim` | 原文直引，必须回原始数据验证 sender | 最高 |
| `artifact` | 统计数据、消息密度、时间分布、文件、转账、链接、行为痕迹 | 中 |
| `impression` | 蒸馏器推断或阅读后的印象 | 最低 |

## 置信度

- 高：多条 verbatim 或 verbatim + artifact 跨时间/跨来源交叉验证，方向一致。
- 中：有 artifact 或少量 verbatim 支撑，但样本不足或情境有限。
- 低：只有 impression，或样本存在明显噪音、冲突、特殊时期影响。

## 写法

每条洞察必须写：

```markdown
### 洞察标题

[洞察] 具体判断。

- 证据：[verbatim] YYYY-MM-DD | source | sender=me(已验证) | "克制引用"
- 证据：[artifact] 消息量、时间分布或文件痕迹
- 置信度：高/中/低
- 状态：新增/修订/待验证/已被 CORR-XXX 推翻
```

## 禁止

- 无证据的洞察不写进自画像。
- `impression` 不能单独作为正式洞察，必须配至少一条 verbatim 或 artifact。
- 关键词统计不能作为关系、情绪、人格定性的唯一证据。
- 未验证 sender 的 verbatim 不能进入正式报告。

## 降级规则

出现以下任一情况，置信度至少降一级：

- 数据来自考试周、假期、生病期、重大事件期。
- 样本只覆盖一个会话或一个很短时间段。
- 关键词有明显多义性或分词噪音。
- 用户自我报告与统计结论冲突。
- sender 只能从预处理摘要中推断，未回原文验证。
