# Action Translation v0.8

Use this reference when the user asks "now what?", wants a weekly plan, wants to turn self-portrait insights into behavior experiments, or needs scripts for communication, boundaries, repair, or decision-making.

Living Mirror should not stop at "this is who you are." It should help the user try a small, reversible next move.

## Translation Rules

- Translate only insights with enough evidence or with the user's explicit confirmation.
- Keep actions small, reversible, and reviewable.
- Do not turn a low-stability insight into a major life decision.
- Separate reflection, experiment, and advice.
- Avoid diagnosis, treatment plans, or crisis intervention. Encourage professional support when needed.

## Action Types

| Type | Use when | Output |
|---|---|---|
| `seven_day_experiment` | The user wants to test a pattern gently. | One tiny action, one observation field, one review date. |
| `conversation_script` | The user needs to say something difficult. | A short script in the user's language, with softer and firmer versions. |
| `boundary_script` | The pattern involves over-giving, delay, resentment, or consent. | Clear yes/no/maybe language. |
| `repair_script` | There was conflict, silence, apology, or re-contact. | Repair attempt, accountability line, and exit condition. |
| `decision_lens` | The user is choosing between options. | Values, energy, relationship cost, reversibility, and evidence check. |
| `environment_adjustment` | The insight depends on context, body, attention, or aesthetic order. | A concrete change to time, space, tools, or rhythm. |

## Seven-Day Experiment Template

```markdown
## 7-Day Experiment: <user-language name>

- Linked insight: INSIGHT-XXX
- Why this is safe to test: <evidence and limits>
- Tiny action: <one action under 20 minutes or one sentence to say>
- When to do it: <time/context>
- What to observe: <body/energy, emotion, relationship response, avoidance, relief>
- What would disconfirm the insight: <counter-evidence>
- Review date: YYYY-MM-DD
- Stop condition: <when to stop or shrink the experiment>
```

## Review Loop

After an experiment:

1. Ask what happened, not whether the user "succeeded."
2. Record supporting evidence and counter-evidence.
3. Update confidence instead of forcing a conclusion.
4. If the user says the action did not fit, write a Correction or rename the pattern.
